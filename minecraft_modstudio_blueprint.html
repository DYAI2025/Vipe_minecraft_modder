Architecture

Process and trust boundaries

Renderer is untrusted UI; Main owns FS and child_process. Preload exposes a minimal safe API.

+------------------------------+
| Renderer (React + Three.js)  |
| - UI, State, Preview, Chat   |
| - calls window.KidMod.*      |
+--------------+---------------+
               |
               | contextBridge API
               v
+------------------------------+
| Preload (bridge.ts)          |
| - ipcRenderer.invoke/on      |
| - exposeInMainWorld KidMod   |
+--------------+---------------+
               |
               | IPC channels
               v
+------------------------------+
| Main (Electron)              |
| - registerIpcHandlers        |
| - helpService (help:search)  |
| - buildService (pipeline:*)  |
| - workspace sandbox utils    |
+--------------+---------------+
               |
               | FS / child_process / temp dirs
               v
+------------------------------+
| Exporter + Gradle + Testbot  |
| - scaffold Fabric template   |
| - generate java/json/assets  |
| - gradlew build              |
| - gradlew runServer + parse  |
+------------------------------+
Build/Test sequence (high level) 
Renderer(BuildDialog) -> pipeline:start(project)
Main(buildService) -> exportProject(tempDir)
 -> runBuild(tempDir) [gradlew build]
 -> runTest(tempDir)  [gradlew runServer + analyzer]
 -> pipeline:progress events -> Renderer

Contracts

Project JSON Spec v1
# Project JSON Spec v1 (Single Source of Truth)

- File: project.json
- schemaVersion: 1
- IDs:
  - EntityId: ^[a-z0-9_]+$
  - ModId:    ^[a-z][a-z0-9_]{1,63}$
  - NamespacedRef: ^[a-z0-9_]+:[a-z0-9_/]+$

Types (v1):
- TextureSpec:
  - preset: { source:"preset", value: rock|wood|gem|metal|grass|sand }
  - dataUri: { source:"dataUri", value:"data:image/png;base64,..." }
  - Recommended: persist always as dataUri.
- BlockSpec: id, name, properties(hardness 0..50, luminance 0..15, transparent), texture
- ItemSpec:  id, name, itemType(gem|sword|tool|food), properties(maxStackSize 1..64, attackDamage 0..50), texture
- RecipeSpec (shaped):
  - pattern 1..3 rows, each 1..3 chars (A-Z0-9 or space)
  - key maps each non-space char to NamespacedRef
  - result { item: NamespacedRef, count 1..64 }

ProjectFile:
- meta: modId, name, version required; optional author/description/createdAt/updatedAt
- blocks/items/recipes: Record<EntityId, Spec>
- Invariant: record key == value.id (or loader fixes + warning)

Semantic validation:
- regex checks
- texture checks
- recipe refs exist or minecraft:*
- result exists or minecraft:*
- recommended: no duplicate ids across blocks/items

ID rename policy:
- Option A (recommended): IDs immutable (UI id read-only), rename via copy+delete
- Option B: explicit rename actions that update refs
IPC / Bridge Spec v1
# IPC / Bridge Spec v1

Principles:
- Renderer untrusted; Main owns FS/child_process.
- JSON-only payloads; validate + sandbox in Main.

Bridge (window.KidMod):
- selectWorkspaceDir()
- saveProject(workspaceDir, project)
- loadProject(workspaceDir)
- searchHelp(query, limit?)
- startBuildPipeline(project) -> runId
- onBuildProgress(cb) -> unsubscribe
- cancelBuildPipeline(runId) optional

Envelope:
- ok true payload OR ok false error {code,message,details?}

Channels:
- dialog:selectWorkspace
- project:save
- project:load
- help:search
- pipeline:start
- pipeline:progress (event)
- pipeline:cancel optional
Exporter Manifest v1
# Exporter Manifest v1

Goal:
- Deterministic Fabric Loom output that builds with gradlew build.

Output (outputDir):
- gradlew + wrapper
- build.gradle/settings.gradle/gradle.properties
- src/main/java/<packagePath>/{MainClass, ModRegistry}.java
- src/main/resources:
  - fabric.mod.json
  - assets/<modId>/{lang, blockstates, models, textures}

Generated:
- lang/en_us.json: block.<modId>.<id>, item.<modId>.<id>
- blockstates/models per manifest
- textures/*.png from dataUri
- Java registry registers blocks, block items, items

Build/test:
- runBuild: gradlew build -> jar in build/libs/*.jar
- testbot: gradlew runServer -> analyzer -> ok/fail

Error Catalog

# Error Catalog v1

Codes:
E_BAD_REQUEST, E_WORKSPACE_REQUIRED, E_PATH_TRAVERSAL, E_NOT_FOUND, E_VALIDATION,
E_IO, E_PIPELINE_RUNNING, E_BUILD_FAILED, E_TEST_FAILED, E_UNKNOWN

UI rules:
- Friendly message always; details optional.

- validateProject -> E_VALIDATION
- FS -> E_IO
- sandbox -> E_PATH_TRAVERSAL

- gradle build fail -> E_BUILD_FAILED
- runServer crash/timeout -> E_TEST_FAILED
